game_map¶

class gamelib.game_map.GameMap(config)¶

Holds data about the current game map and provides functions useful for getting information related to the map.

game_map[x, y] will return a list of Units located at that location, or an empty list if there are no units at the location

* config

Contains information about the game

Type:JSON
* ARENA_SIZE

The size of the arena.

Type:int
* HALF_ARENA

Half of the size of the arena.

Type:int
* TOP_RIGHT

A constant that represents the top right edge

Type:int
* TOP_LEFT

A constant that represents the top left edge

Type:int
* BOTTOM_LEFT

Hidden challenge! Can you guess what this constant represents???

Type:int
* BOTTOM_RIGHT

A constant that represents the bottom right edge

Type:int
add_unit(unit_type, location, player_index=0)¶

Add a single GameUnit to the map at the given location.

Parameters:
  • unit_type (*) – The type of the new unit
  • location (*) – The location of the new unit
  • player_index (*) – The index corresponding to the player controlling the new unit, 0 for you 1 for the enemy

This function does not affect your turn and only changes the data stored in GameMap. The intended use of this function is to allow you to create arbitrary gamestates. Using this function on the GameMap inside game_state can cause your algo to crash.

get_edge_locations(quadrant_description)¶

Takes in an edge description and returns a list of locations.

Parameters:quadrant_description (*) – A constant corresponding to an edge. Valid quadrant descriptions are * GameMap.TOP_RIGHT * GameMap.TOP_LEFT * GameMap.BOTTOM_RIGHT * GameMap.BOTTOM_LEFT
Returns:A list of locations corresponding to the requested edge
get_edges()¶

Gets all of the edges and their edge locations

Returns:A list with four lists inside of it of locations corresponding to the four edges. [0] = top_right, [1] = top_left, [2] = bottom_left, [3] = bottom_right.
